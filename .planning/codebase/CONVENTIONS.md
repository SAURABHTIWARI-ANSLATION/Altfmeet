# Coding Conventions

- **Module System**: Using ES Modules (`import`/`export`) as per `package.json` (`"type": "module"`).
- **Style**: Standard JavaScript formatting.
- **Frontend**: Separation of concerns across `media.js`, `signaling.js`, and `chat.js`.
- **Backend**: Modularized by feature (e.g., `media/`, `signaling/`).
- **Naming**: camelCase for variables/functions, snake_case often used for directory names or DB fields (implied).
